## 三、数据模型

### Redis 存储

```
Key:   cart:{userId}      类型: Hash
Field: {productId}        Value: CartItemEntry JSON
TTL:   当天 23:59:59（每次写操作刷新）
```

CartItemEntry:
```json
{
  "quantity": 2,
  "priceSnapshot": 9.90,
  "titleSnapshot": "红富士苹果 5斤装",
  "imageSnapshot": "/api/file/images/abc123.jpg"
}
```

| id | bigint | 主键自增 |

| quantity | int | 数量 |
| price_snapshot | decimal | 价格快照 |
| title_snapshot | varchar | 标题快照 |
| image_snapshot | varchar | 主图 URL 快照 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

### 4.1.1 OmsCartServiceImpl#add 并发安全详细流程（Redisson 锁版）

```
POST /api/cart/add { productId, quantity }
        |
        v
 OmsCartServiceImpl.add(userId, param)
        |
        v
 lockKey = "cart:lock:" + userId + ":" + productId
 RLock lock = redissonClient.getLock(lockKey)
        |
        v
 lock.tryLock(等待3s, 持锁5s)
        |
   +-----------+------------+
   |                        |
 拿到锁                  未拿到锁（3s内)
   |                        |
   v                        v
[商品校验]          throw IllegalStateException
 · selectById(productId)    "操作频繁，请稍后重试"
 · product == null -> 商品不存在
 · status != 1   -> 商品已下架
   |
   v
[读 Redis 当前数量]（锁保护内，串行安全）
 CartRedisHelper.getItem(userId, productId)
 currentQty = existing != null ? existing.quantity : 0
 newQty = currentQty + addQty
   |
   v
[库存校验]
 newQty > product.stock -> 库存不足
   |
   v
[限购校验]
 newQty > limitPerUser  -> 超出限购数量
   |
   v
 extractFirstImage(product.images) -> imageSnapshot
   |
   v
 CartItemEntry(newQty, price, title, imageSnapshot)
   |
   +------------------+
   |                  |
   v                  v
Redis               @Async
setItem()          asyncUpsert()
Hash PUT           MySQL upsert
TTL 刷新           重试 3 次
   |
   v
 finally 块
 locked && lock.isHeldByCurrentThread()
        -> lock.unlock()
   |
   v
 return（Controller 层包装 Result.success）
```

**并发安全关键点说明：**

| 要点 | 说明 |
|---|---|
| 锁粒度 | `userId + productId` 双维度，不同商品互不阻塞 |
| tryLock 超时 | 等待最多 3 秒，防止线程无限堆积 |
| 持锁上限 | 最多持锁 5 秒，防止业务异常导致死锁 |
| isHeldByCurrentThread | finally 中判断后再 unlock，防止解锁他人的锁 |
| InterruptedException | 捕获后恢复中断状态，不吞异常 |
| 串行保证 | 同一用户同一商品的并发请求在锁内排队执行，数量累加结果精确 |

### 4.4 删除 / 清空流程

```
删除单项: DELETE /api/cart/remove { productId }
  -> Redis.removeItem() -> @Async asyncDelete()

  -> Redis.clear() -> @Async asyncClear()
  -> 前端 items = [], selected 全清
```

## 五、对标大厂的问题与差距

### 5.1 数据一致性问题

| # | 问题 | 当前实现 | 风险 | 大厂方案 | 状态 |
|---|---|---|---|---|---|
| 1 | Redis 与 MySQL 弱一致 | @Async 异步落库失败仅打日志 | 高 | MQ + 死信队列兜底 | ⏳ 待完成 |
| 2 | 并发加购数量不准 | Redisson 分布式锁保护「读-校验-写」，锁粒度 userId+productId | 中 | Lua 脚本原子操作 / Redisson 锁 | ✅ 已完成 |
| 3 | 价格快照不刷新 | 加购时记录一次，后续不更新 | 中 | 结算时重新比价，变动提示用户 | ⏳ 待完成 |
| 4 | TTL 当天过期 | 每天 23:59:59 失效 | 低 | 7~30 天 TTL，活跃用户自动续期 | ⏳ 待完成 |

### 5.2 业务功能缺失

| # | 缺失 | 影响 | 参考方案 |
|---|---|---|---|
| 1 | 无 SKU 规格 | 无法区分商品不同规格 | 以 (productId, skuId) 为唯一键 |
| 2 | 无游客购物车 | 未登录无法加购 | localStorage 存储，登录后 merge |
| 3 | 选中状态不持久 | 刷新后丢失 | Redis 持久化或 sessionStorage |
| 4 | 无促销关联 | 看不到满减优惠 | 接口返回命中促销规则和优惠金额 |
| 5 | 无失效自动清理 | 下架商品堆积 | 定时任务扫描并通知用户 |
| 6 | 无实时库存推送 | 库存变化不及时感知 | WebSocket / SSE 推送 |
| 7 | 无跨设备同步 | 手机/PC 购物车不一致 | 服务端为唯一数据源 + WS |
| 8 | 无凑单推荐 | 结算无差价提示 | 根据金额推荐凑满减商品 |

### 5.3 性能问题

| # | 问题 | 当前实现 | 优化方向 |
|---|---|---|---|
| 1 | N+1 查询 | buildVOList 逐条查 pms_product | selectBatchIds() 批量查询 |
| 2 | 商品信息无缓存 | 每次实时查 MySQL | 热点商品信息缓存到 Redis |
| 3 | 库存校验走 DB | add 时同步查数据库 | 热点库存放 Redis，减少 DB 压力 |

## 六、后续优化路线图

### 阶段一：稳定性（高优先级）

- [ ] **N+1 修复**：`buildVOList` 改为 `selectBatchIds` 批量查商品
- [ ] **异步落库可靠性**：引入 RocketMQ/Kafka，失败投死信队列，告警通知
- [ ] **并发安全**：加购用 Lua 脚本原子操作
  ```lua
  local qty = redis.call('HGET', key, productId)
  local newQty = (qty or 0) + addQty
  -- 校验库存后
  redis.call('HSET', key, productId, newQty)
  ```

### 阶段二：功能完善（中优先级）

- [ ] **游客购物车**：前端 localStorage 存储，登录后调 `/api/cart/merge` 合并
- [ ] **SKU 支持**：表结构加 sku_id 字段，唯一索引改为 (user_id, product_id, sku_id)
- [ ] **选中状态持久化**：Redis Hash 单独存 selected 状态字段
- [ ] **结算价格二次校验**：下单时对比快照价与当前价，差异超过阈值提示用户

### 阶段三：体验提升（低优先级）

- [ ] **TTL 延长**：改为 30 天，用户活跃时自动续期
- [ ] **促销信息展示**：购物车接口聚合满减/优惠券信息
- [ ] **实时库存提示**：SSE 长连接推送库存不足预警
- [ ] **凑单推荐**：根据已选金额推荐差价最小的凑满减商品
